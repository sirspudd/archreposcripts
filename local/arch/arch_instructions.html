Overview

Qt can run quite nicely on the Pi(s), but the packaged versions shipped with most distros use Mesa out of the box, and people tend to run X11 on the pi for graphical purposes, and that seems like a damn shame given the GPU is decent and can actually do a fair amount. The point of this exercise is to make 2 packages conveniently available, one for the Raspberry Pi 2 and one for the host machine, which once installed allow you to cross compile for the Raspberry Pi 2 from your Arch host machine. The AUR recipes used to generate the installed packages are here (in case you are into that kind of thing):

    Instructions

  • Install Arch on a fresh sd card following the following instructions
  • Boot the Raspberry Pi 2
  • Download and run the following script on the RPI 2
  • I personally normally do this by allowing root login via ssh and then grabbing the script via curl
  • curl http://s3.amazonaws.com/spuddrepo/arch/setup_qpi.sh > setup_qpi.sh
  • bash setup_qpi.sh
  • I would strongly recommend increasing the video ram to 512 in /boot/config.txt (The compositor alone requires more than the default 64 due to the sexy background wave effect)
  • Run this script on your Arch host machine
  • In order for mdns hostname resolution to work, you have to enable this which I can't safely automate
  • Mount your Arch sysroot on your host (recommend NFS, mounting "/" under /mnt/pi)
  • You are now set to use Qt Creator to develop and deploy to your Raspberry Pi 2
  • Please note, you might have to explicitly "test" the device in the device page of Qt Creator in order for deployment to work. (pessimistic)
